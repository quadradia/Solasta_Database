# Proxy ID Implementation – GBDC_MAIN Database

## Overview

Every major entity in the NOAH/GBDC platform is identified externally by a **Proxy ID** — a structured, human-readable string that encodes the entity type, the creation time period, and a unique identifier. Proxy IDs are used across API contracts, cross-service communication, and security lookups to avoid exposing raw integer primary keys.

---

## Format

```
{Prefix}:{PartKey}:{GUID}
```

| Segment | Description | Example |
|---|---|---|
| **Prefix** | 4-character uppercase code identifying the entity type and subtype | `AUWM` |
| **PartKey** | Hexadecimal encoding of the year+month of creation (trimmed) | `31710` |
| **GUID** | Standard `UNIQUEIDENTIFIER` (36 chars) generated at insert time | `6B7F44A9-35F8-44AB-B34A-FEBD29607D2B` |

**Full example:**
```
AUWM:31710:6B7F44A9-35F8-44AB-B34A-FEBD29607D2B
```

---

## PartKey Encoding

The `PartKey` is generated by `[UTL].[fxGetPartKey]()` and encodes the current year and month as a hexadecimal string, making it compact and time-sortable.

**Algorithm:**
1. Capture the current `YEAR` and `MONTH` from `GETDATE()`.
2. Left-pad the month to 2 digits and concatenate: `YYYYMM` (e.g., `202605` for May 2026).
3. Cast the string to `INT`.
4. Convert to `VARBINARY(4)` then to a hex string via `CONVERT(..., 1)`.
5. Strip the leading `0x000` prefix, leaving a 5-character hex string.

**Example for May 2026:**
```
Year=2026, Month=05  →  202605  →  0x000316BD  →  316BD
```

**SQL Source:** `FUNCTIONS/UTL/fxGetPartKey.sql`

```sql
CREATE FUNCTION [UTL].[fxGetPartKey] ()
RETURNS VARCHAR(6)
AS
BEGIN
    DECLARE @Year    VARCHAR(4)  = CAST(YEAR(GETDATE()) AS VARCHAR(4));
    DECLARE @Month   VARCHAR(2)  = RIGHT('0' + CAST(MONTH(GETDATE()) AS VARCHAR(2)), 2);
    DECLARE @StrValue VARCHAR(10) = @Year + @Month;
    DECLARE @Number   INT         = CAST(@StrValue AS INT);
    DECLARE @HexValue VARCHAR(10) = CONVERT(VARCHAR(10), CONVERT(VARBINARY(4), @Number), 1);
    RETURN REPLACE(@HexValue, '0x000', '');
END
```

## Database Objects

### Generation

| Object | Schema | Type | Purpose |
|---|---|---|---|
| `fxGetPartKey` | `UTL` | Scalar Function | Generates the hex year+month PartKey for the current date |
| `fxGeneratePrefixParts` | `MAC` | Table-Valued Function | Returns `(Prefix, PartKey)` for a given TypeName + TypeId |
| `fxGeneratePrefixProxy` | `MAC` | Scalar Function | Returns `Prefix:PartKey:` (the GUID is appended by the caller) |

### Parsing & Lookup

| Object | Schema | Type | Purpose |
|---|---|---|---|
| `fxGetPrefixTypeIdFromProxy` | `MAC` | Table-Valued Function | Splits a proxy into its components and resolves TypeId, TypeName, TableName |
| `fxGetEntityInfoByProxy` | `ACC` | Table-Valued Function | Resolves an entity proxy to EntityID, FirstName, LastName, and proxy components |
| `vwTableTypesAndPrefixes` | `MAC` | View | Central registry of all Prefix → TypeId/TypeName/TableName mappings |

## Generation Flow

Below is the step-by-step process used when a new entity is created:

```
1. Caller determines TypeName (e.g., 'Transaction') and TypeId (e.g., 3)
        ↓
2. [MAC].[fxGeneratePrefixParts](@TypeName, @TypeId)
   - Queries the appropriate type table for the Prefix
   - Calls [UTL].[fxGetPartKey]() for the current PartKey
   - Returns (Prefix, PartKey)
        ↓
3. [MAC].[fxGeneratePrefixProxy](@TypeName, @TypeId)
   - Calls fxGeneratePrefixParts internally
   - Returns the string: "Prefix:PartKey:"
        ↓
4. Caller appends CAST(NEWID() AS NVARCHAR(50))
   - Final Proxy = "Prefix:PartKey:GUID"
        ↓
5. Proxy is stored in the entity's table row (e.g., [GBD].[Transactions].Proxy)
```

**Example from `spTransactionP2PStart`:**
```sql
DECLARE @Proxy NVARCHAR(50) = (SELECT TOP(1) [MAC].[fxGeneratePrefixProxy]('Transaction', @TransactionTypeID));
SET @Proxy = @Proxy + CAST(NEWID() AS NVARCHAR(50));
```

**Example from `spFinishKycIndividual` (manual assembly):**
```sql
SELECT @EntityPrefix = Prefix, @EntityPartKey = PartKey
FROM [MAC].[fxGeneratePrefixParts]('Entity', @EntityTypeId);

SET @EntityGuid = NEWID();
SET @EntityProxy = @EntityPrefix + ':' + @EntityPartKey + ':' + CAST(@EntityGuid AS NVARCHAR(36));
```

## Parsing Flow

To resolve a proxy back to its components:

```
1. Input: "AUWM:31710:6B7F44A9-35F8-44AB-B34A-FEBD29607D2B"
        ↓
2. [MAC].[fxGetPrefixTypeIdFromProxy](@Proxy)
   - Extracts Prefix  = "AUWM"
   - Extracts PartKey = "31710"
   - Extracts GUID    = "6B7F44A9-35F8-44AB-B34A-FEBD29607D2B"
   - Looks up [MAC].[vwTableTypesAndPrefixes] WHERE Prefix = "AUWM"
   - Returns: (Prefix, PartKey, GuidIdentifier, TypeId, TableName, TypeName)
        ↓
3. Caller uses TypeId + PartKey + full Proxy for validated lookups
```

**Example — Entity lookup:**
```sql
SELECT * FROM [ACC].[fxGetEntityInfoByProxy]('AUWM:31710:6B7F44A9-35F8-44AB-B34A-FEBD29607D2B');
```

**Example — Security lookup by audience + GUID:**
```sql
EXEC [SEC].[spGetEntityProxyByAudienceAndGuid]
    @Audience = 'apiBaaS',
    @Sub       = '6B7F44A9-35F8-44AB-B34A-FEBD29607D2B';
```
> This proc maps an API audience name to a known `Prefix` (e.g., `apiBaaS` → `QDIP`) and then searches `ACC.Entities` using a `LIKE '%' + GUID` pattern.

## Storage Conventions

- The `Proxy` column is stored as `NVARCHAR(100)` on most entity tables.
- The `PartKey` segment is also stored as a **separate column** (`VARCHAR(5)`) on entities (e.g., `ACC.Entities.PartKey`, `GBD.Transactions.PartKey`) to allow indexed lookups without string parsing at query time.
- Validation joins combine both `PartKey` and `TypeId` to avoid cross-type collisions:

```sql
INNER JOIN [MAC].[fxGetPrefixTypeIdFromProxy](@EntityProxy) AS EBI
ON  (EBI.PartKey = EB.PartKey)
AND (EBI.TypeId  = EB.EntityTypeId)
WHERE (EB.Proxy = @EntityProxy)
```

## Adding a New Entity Type

To support a new entity type in the Proxy ID system:

1. **Add a type table** (e.g., `[SCHEMA].[FooTypes]`) with a `Prefix` column containing a unique 4-character code.
2. **Register it in `MAC.vwTableTypesAndPrefixes`** — add a `UNION` block that selects from the new type table with the correct `TypeName` and `Schema` values.
3. **Register it in `MAC.fxGeneratePrefixParts`** — add an `ELSE IF (@TypeName = 'Foo')` branch that queries the new type table.
4. Generation and parsing will work automatically via the existing functions once both registrations are in place.

## Key Design Principles

- **No integer PKs in public contracts.** External systems only ever see the Proxy string.
- **Type safety.** The Prefix encodes the subtype; the TypeId validates it on lookup. A proxy for a Transaction cannot be used to look up a Person.
- **Time-partitioned.** The PartKey embeds creation year+month, enabling range-based queries and data lifecycle management without parsing GUIDs.
- **Decentralized prefix definition.** Prefixes live in domain-owned type tables (not a single global enum), keeping schema boundaries clean.
- **Idempotent lookups.** Storing the full Proxy string on the row allows direct equality lookups; storing PartKey separately enables indexed partition filtering.
